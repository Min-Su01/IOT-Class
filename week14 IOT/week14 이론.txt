Wifi 구성
-IEEE Working Group에서 표준화 작업
-AP(유뮤선 공유기)와 STA(노트북, 스마트폰등)으로 구성

Wifi Service Set
-BSS (AP가 없으면 ad hoc모드, AP가 있으면 infrastructure모드)
-ESS (두 개 이상의 BSS들이 모여서 구성)

MAC방식
-PCF( Option, 중앙집중식 Polling 방식 사용)
-DCF( CSMA/CA 사용. Station에서 사용)

CSMA-CA(아래 CD와 대조하기)
-CS: 네트워크가 현재 사용 중인지 알아냄
-MA: 네트워크가 비어있으면 누구든 사용 가능
-CA: 충돌 회피

충돌(컬리젼)
-두 개 이상의 노드가 같은 시간대에 같은 노드에게 패킷을 전송
-수신 축은 데이터를 제대로 읽을 수 있게 됨.

히든 터미널 프라블럼(숨겨진 노드 문제)
-교집합 A네트워크는 B,C와 통신 가능, but B,C는 서로 통신을 감지하지 못함

RTS-CTS-DATA-ACK 방식
-송식 측에서 RTS전송 후, 수신 측에서 CTS 전송
-송신 측에서 CTS를 전송 받지 못하면, 일정 시간 대기 후, RTS를 다시 전송

CSMA-CD
-이더넷에 쓰이는 채널 접근 방식
-송신 중 충돌이 감지되면, 전송을 중지하고 랜덤한 시간 동안 기다리고, 다시 전송 시도
-유선에서는 구현이 쉬우나, 무선에서 어려움을 겪을 수 있다.

위 CSMA-CA의 CSMA-CD와 비교한 장단점 
-CA의 장점: 저 가격으로 구현
-CA의 단점: 데이터 전송이 많이 지연될 수 있음


-ZigBee스택-
스택: 프로토콜, 소프트웨어 관한 집합체라 볼 수 있음
지그비 스택: 지그비에서 정의한 프로토콜들의 구현
지그비의 CSMA-CA : RTS,CTA를 사용하지 않음


물리(PHY)계층
IEEE PHY 계층 특징
-무선으로 직접 데이터 주고 받는 계층
-3개의 밴드에 27개의 채널 사용
-DSSS사용
-빠른 응답 지원, 전력소모 최소화하는 전력 관리
(물리 계층 각각 특징의 자세한 내용은 ppt참조)

MAC 계층
-세 종류의 기기가 정의됨: 1.네트워크 코디네이터, 2.전기능 기기 3.축소기능기기
-MAC의 선택 사양 :1. 비콘 없는 통신, 2.비콘 사용 통신
(비콘 없는 통신 특) : 1.Non-slotted CSMA-CA통신, 2.수신 패킷 성공을 위해 확인 응답
(비콘 사용 통신 특) : 1.Slotted CSMA-CA통신, 슈퍼 프레임 사용, 2.백오프 하는 시간을 슬롯 단위로 한다.

슈퍼프레임 구조
-최대 16개의 슬롯으로 구성
-슈퍼프레임은 비콘, CAP,CFP으로 구성
-슈퍼프레임은 최소 15ms에서 최소 245초 가능
-항상 비콘으로 시작(비콘은 PAN코디네이터 (네트워크코디네이터)가 송신)

CAP 경쟁구간
-슬롯 단위의 시간에 맞추어 송수신

CFP 경쟁 없이 송수신 하는 구간
-코디네이터가 슬롯을 예약하여 지그비 기기들이 그 슬롯 시간에 송수신

비콘
-비콘에 따라 시간 동기화를 하여 슬롯 단위 송수신 가능
-GTS가 어느 기기에 할당되었는지 알려줌

비활성화 구간
-지그비 기기의 전력 사용을 적게하여 기기 수명을 늘리기 위해 비활성화 구간 사용

MAC 프레임의 종류(타입) 4가지
-비콘 프레임, 데이터 프레임, 확인 프레임, MAC명령 프레임

데이터 프레임
-데이터의 송수신에 사용
-Frame control필드: Frame type, 주소 형식 등 Frame에 관한 정보
-Sequence Number 필드: Frame의 일렬 번호 -> 전송 확인하기 위해 사용
-Payload 필드 : 실제 Data가 실리는 곳
-Frame check Sequence 필드 : Frame의 error를 check


분산 주소 할당 기법
-16비트 어드레스 할당 방법
-지그비 장치를 가진 노드가 지그비 네트워크에 참여할 때 이 노드의 부모 노드가 정해진 식에 따라 부여
-지그비 라우터는 모두 자신의 노드에게 주소 할당을 할 수 있음 -> 분산방식
